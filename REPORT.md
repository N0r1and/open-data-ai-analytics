### 1. Мета роботи
Створити базову структуру проєкту для аналізу відкритих даних (`open-data-ai-analytics`), налаштувати систему контролю версій Git, відпрацювати створення гілок (feature branches), злиття через Pull Requests, розв'язання merge-конфліктів та тегування релізів.

---

### 2. Виконані завдання

**1. Ініціалізація репозиторію:**
Створено локальний та віддалений репозиторії `open-data-ai-analytics`. Налаштовано структуру папок (`data`, `notebooks`, `src`, `reports/figures`) та файл `.gitignore` для виключення Python-кешу, віртуального середовища та папки `data/raw/` з великими файлами даних. Заповнено `README.md` з описом проєкту (аналіз індексів цін на житло в Україні, джерело — data.gov.ua) та трьома гіпотезами: вплив повномасштабного вторгнення (2022) на ціни, порівняння первинного і вторинного ринків, дослідження сезонності.

**2. Робота з гілками (Git Flow):**
- Створено гілку `feature/data_load` із скриптом завантаження та очищення датасету (`src/data_load.py`). Гілку змерджено напряму у `main`.
- Створено паралельні гілки `feature/data_quality_analysis` (перевірка якості даних: пропуски, дублікати, аномалії) та `feature/data_research` (статистичний аналіз: t-тест, ANOVA, лінійна регресія). Злиття обох гілок відбулося через Pull Requests (#1 та #2) на GitHub з описом виконаних змін.
- Додано гілку `feature/visualization` з кодом для побудови 4 графіків: загальний тренд, порівняння ринків, boxplot сезонності, теплова карта.

**3. Вирішення конфліктів:**
Штучно створено merge-конфлікт у файлі `README.md` шляхом одночасної зміни секції гіпотез у двох гілках (`conflict-branch-A` та `conflict-branch-B`). Конфлікт успішно розв'язано локально — залишено фінальний коректний варіант тексту.

**4. Реліз:**
Додано `CHANGELOG.md` з описом всіх змін проєкту. Фінальний стан зафіксовано анотованим тегом `v0.1.0`.
